GrassRootsBBS 運用マニュアル

このドキュメントは、GrassRootsBBSのセットアップ、管理、利用方法について説明します。

1. Installation & Setup / インストールとセットアップ

このセクションでは、サーバーを起動してBBSを運用可能にするまでの手順を説明します。

1.1. 環境変数の設定 (.env)

プロジェクトのルートディレクトリに .env ファイルを作成し、システム管理者（シスオペ）のアカウント情報を設定します。

# .env ファイルの例
GRASSROOTSBBS_SYSOP_ID=your_sysop_id
GRASSROOTSBBS_SYSOP_PASSWORD=your_strong_password
GRASSROOTSBBS_SYSOP_EMAIL=your_email@example.com
  • GRASSROOTSBBS_SYSOP_ID: シスオペのログインIDです。
  • GRASSROOTSBBS_SYSOP_PASSWORD: シスオペの初期パスワードです。ログイン後、速やかに変更してください。
  • GRASSROOTSBBS_SYSOP_EMAIL: シスオペのメールアドレスです。オンラインサインアップの通知などに使用されます。

1.2. ドメイン名の設定 (config.toml)

Passkey認証やPush通知などの機能を利用するには、BBSにアクセスするためのドメイン（またはIPアドレス）を設定する必要があります。setting/config.toml ファイルを編集してください。

# setting/config.toml

[webapp]
ORIGIN = "http://your.bbs.domain:5000"

1.3. VAPIDキーの生成 (Push通知用)

Push通知機能を使用するには、VAPIDキーペアが必要です。以下のコマンドを実行してキーを生成してください。

python3 tools/generate_vapid_keys.py

このコマンドにより、プロジェクトのルートディレクトリに private_key.pem と public_key.pem が作成されます。

  1. 生成された private_key.pem は、プロジェクトのルートディレクトリにそのまま配置しておきます。
  2. public_key.pem の中身（文字列）をコピーし、setting/config.toml ファイルの [push] セクションにある VAPID_PUBLIC_KEY の値として貼り付けます。

1.4. サーバーの起動

設定が完了したら、Docker Composeを使ってBBSを起動します。

docker-compose up --build -d

初回起動時に、データベースのテーブル作成と、.env で設定したシスオペアカウントの作成が自動的に行われます。

2. For System Operators / 管理者向けマニュアル

このセクションでは、BBSの管理と運営に関するタスクを説明します。

2.1. 管理画面へのアクセス

シスオペは、以下のURLからWebベースの管理画面にアクセスできます。

  • http://<サーバーのIPアドレス>:5000/admin

ログイン後、ユーザー管理、掲示板管理、システム設定など、BBSのあらゆる側面を直感的に管理できます。

2.2. 掲示板とチャットルームの構成

Webターミナル上でのメニュー構造は、以下のYAMLファイルを編集することで定義します。

  • 掲示板: setting/bbs_mode3.yaml
  • チャット: setting/chatroom.yaml

2.3. 掲示板の作成と編集

YAML ファイルに掲示板を追加するだけでは不十分です。管理画面から、実際に掲示板のデータを作成する必要があります。

  1. 管理画面にログインし、「Board Management」に移動します。
  2. 「+ Add New Board」ボタンをクリックします。
  3. YAMLファイルで定義した id と同じ「Shortcut ID」や、掲示板名、権限などを設定して保存します。

2.4. ログインメッセージの変更

管理画面の「System Settings」ページから、ユーザーがログインした際に表示されるメッセージを編集できます。

2.5. バックアップ

管理画面の「Data Management」ページから、手動でバックアップを作成したり、古いバックアップを復元・削除したりできます。また、自動バックアップのスケジュールもこの画面で設定可能です。

# setting/config.toml
[backup]
max_backups = 7 # 自動クリーンアップで保持するバックアップの最大数

3. For Users / ユーザー向けマニュアル

このセクションでは、BBSの基本的な使い方を説明します。

3.1. BBSへのアクセス

Webブラウザから以下のURLにアクセスしてBBSを利用します。

  • http://<サーバーのIPアドレス>:5000

BBSのログイン画面が表示されたら、ユーザーIDとパスワード、またはPasskeyを使用してログインしてください。

3.2. 主な機能

掲示板 (BBS)

  • 読み書き: r で記事一覧を読み、w で新しい記事を投稿します。
  • 探索リスト: 記事一覧画面で @ を押すと、その掲示板を「探索リスト」に追加/削除できます。トップメニューの n (新アーティクル探索) などで、リストに登録した掲示板だけを効率的に巡回できます。
  • ファイル添付: 記事投稿時、本文入力後にファイルを添付できます。添付できるファイルの種類やサイズは、管理画面の各掲示板編集ページ、または config.toml で設定可能です。

チャット (Chat)

  • 入室: メニューからチャットルームを選択して入室します。
  • コマンド: チャット中に !? を入力すると、利用可能なコマンド（退室、オンラインメンバー表示など）のヘルプが表示されます。

Passkey (パスワードレス認証)

Webターミナルでログイン後、ユーザー設定 (u コマンド) の「Passkey 管理」から、顔認証や指紋認証、USBセキュリティキーなどを使ってログインするためのPasskeyを登録できます。登録後は、パスワードなしで安全にログインできるようになります。
