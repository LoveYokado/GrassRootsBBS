# GrassRootsBBS 運用マニュアル

このドキュメントは、GrassRootsBBS をホストとして運用する方向けの基本的な使い方を説明します。

## 1. 初回起動とシスオペ設定

BBS を初めて起動する前に、いくつかの重要な設定を行う必要があります。

### 1.1. 環境変数の設定

プロジェクトのルートディレクトリにある `.env` ファイルを編集（または作成）し、シスオペ（システム管理者）のアカウント情報を設定します。

```bash
# .env ファイルの例
GRASSROOTSBBS_SYSOP_ID=your_sysop_id
GRASSROOTSBBS_SYSOP_PASSWORD=your_strong_password
GRASSROOTSBBS_SYSOP_EMAIL=your_email@example.com
```

- `GRASSROOTSBBS_SYSOP_ID`: シスオペのログイン ID です。
- `GRASSROOTSBBS_SYSOP_PASSWORD`: シスオペの初期パスワードです。ログイン後、速やかに変更してください。
- `GRASSROOTSBBS_SYSOP_EMAIL`: シスオペのメールアドレスです。オンラインサインアップの通知などに使用されます。

### 1.2. ドメイン名の設定 (任意)

プッシュ通知や Passkey などの機能で外部連携を行う場合、`setting/config.toml` ファイル内のドメイン名を設定する必要があります。

```toml
# setting/config.toml

[webapp]
ORIGIN = "http://your.bbs.domain:5000" # WebターミナルのURL
```

### 1.3. サーバーの起動

設定が完了したら、以下のコマンドで BBS を起動します。

```bash
docker-compose up --build -d
```

初回起動時に、データベースのテーブル作成と、`.env`で設定したシスオペアカウントの作成が自動的に行われます。

## 2. BBS へのアクセス方法

Web ブラウザから以下の URL にアクセスします。

- `http://<サーバーのIPアドレス>:5000`

この Web ターミナルが、BBS への主要なアクセス方法となります。
BBS のログイン画面が表示されたら、ユーザー ID とパスワード、または Passkey を使用してログインしてください。

## 3. シスオペが最初にやること

BBS を起動したら、シスオペとしてログインし、BBS の「顔」となる部分をカスタマイズしましょう。

### 3.1. 掲示板とチャットルームの構成

BBS のメニュー構造は、以下の YAML ファイルで定義されています。

- **掲示板**: `setting/bbs_mode3.yaml`
- **チャット**: `setting/chatroom.yaml`

これらのファイルを編集して、カテゴリや掲示板、チャットルームの名前や階層を自由に変更してください。

### 3.2. 掲示板の作成

YAML ファイルに掲示板を追加するだけでは不十分です。シスオペメニューから、実際に掲示板のデータを作成する必要があります。

1.  トップメニューで `s` を入力し、シスオペメニューに入ります。
2.  `MKBD` (掲示板作成) を選択します。
3.  YAML ファイルで定義した `id` (ショートカット ID) や、掲示板名、権限などを設定します。

### 3.3. ログインメッセージの変更

シスオペメニューの `LGMG` (ログインメッセージ変更) から、ユーザーがログインした際に表示されるメッセージを編集できます。

## 4. 主な機能の使い方

### 4.1. 掲示板 (BBS)

- **読み書き**: `r` で記事一覧を読み、`w` で新しい記事を投稿します。
- **探索リスト**: 記事一覧画面で `@` を押すと、その掲示板を「探索リスト」に追加/削除できます。トップメニューの `n` (新アーティクル探索) などで、リストに登録した掲示板だけを効率的に巡回できます。
- **ファイル添付**: 記事投稿時、本文入力後にファイルを添付できます。添付できるファイルの種類やサイズは `config.toml` で設定可能です。

### 4.2. チャット (Chat)

- **入室**: メニューからチャットルームを選択して入室します。
- **コマンド**: チャット中に `!?` を入力すると、利用可能なコマンド（退室、オンラインメンバー表示など）のヘルプが表示されます。

### 4.3. プッシュ通知

1.  Web ターミナルでログインします。
2.  右上のハンバーガーメニューから設定画面を開きます。
3.  「Push Notifications」セクションの「Enable Notifications」ボタンをクリックし、ブラウザからの許可を求められたら「許可」します。

これにより、`chatroom.yaml` で `push: True` に設定されたチャットルームに誰かが入室した際に、デスクトップ通知が届くようになります。通知をクリックすると、そのチャットルームに直接移動できます。

### 4.4. Passkey (パスワードレス認証)

Web ターミナルでログイン後、ユーザー設定 (`u` コマンド) の「Passkey 管理」から、顔認証や指紋認証、USB セキュリティキーなどを使ってログインするための Passkey を登録できます。登録後は、パスワードなしで安全にログインできるようになります。
