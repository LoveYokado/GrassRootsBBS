# GrassRootsBBS 説明書

## 本家 Big-model との違い

### サーバ設定(config.toml)

昔の BBS のように少数回線にする場合、NORMAL_SSH_PORT_COUNT を回線数、MAX_CONCURRENT_NORMAL_SSH_CLIENTS を 1 にしてください。

#### 3 回線ホストの再現

NORMAL_SSH_PORT_COUNT = 3 # 通常接続のポート数  
MAX_CONCURRENT_NORMAL_SSH_CLIENTS = 1 # 通常接続の同時接続可能クライアント数(0 で無制限)

ただし、ローカルログインができないのでシスオペも 1 回線埋めてしまうことに注意してください。

### ユーザでの操作

Big-modelの直感的なメニューモードを忠実に再現し、電報機能やシグ探索の改善を計画中です。  
今後、異なる部分をさらに潰し、本家に近づける方向で開発を進めます。

### シスオペでの操作

この部分はかなり変更が加えられています。
* 動作環境がMS-DOSではない
* モデム接続ではなくSSH接続である
* 接続数の上限がない
* 住所や電話番号を聞く時代ではなくなっている
* 階層化構造、ホスト設定、データベース構造がすべて違う。
以上がその理由なのですが、UIの部分ではあまり違和感の出ないように当時の雰囲気を残す形を取ろうと思います。


## 謝辞

このプロジェクトは、かつて隆盛を極めた草の根 BBS 文化と、その中で多くのユーザーに愛されたソフトウェア「BIG-Model」に大きなインスピレーションを受けています。
私自身シスオペをしていた時期もあり、色々な方と出会い、今の私の結構な範囲の基礎を作ってくれたのがこの時代でした。
主に文字のみのコミュニケーションながら、毎夜毎夜電話料金を嵩ませながらチャットする人、日に何度も巡回して掲示板に怒涛の書き込みをする人、他にも喧嘩したり結婚したりオフの飲み会で羽目を外しすぎたり、今のインターネットにはない独特な空気が流れていたのを覚えている人もいると思います。
今更文字主体のBBSを盛り上げるのは無理だと思います。が、ひとつの思い出として今の時代でも少しの手間で誰でもパソコン通信のホストプログラムを立ち上げられる。追体験は無理でも、こんなことをやってたんだって知ってもらえれば幸いです。

作成にあたってBIG-Model の直感的なUIは、本プロジェクトの操作感を実現する上で多大な影響を与えてくれました。もちろん思い出の中のBig-modelが使われていた草の根BBSの印象も強かったんだと思います。

この場を借りて、Big-modelの操作性に類似したGR-BBSの作成に大歓迎とまで仰ってくれたネットコンプレックス株式会社 代表取締役 川村清様、および BIG-Model の開発・運営に携わられた全ての関係者の皆様、シスオペとしてBig-modelを育てていってくれた皆様、あの当時一緒い遊んでくれたみんなに心より感謝申し上げます。

過去の偉大なソフトウェアとその遺産に敬意を表し、本プロジェクトが新たな価値を生み出す一助となれば幸いです。
